#### 6-1. 코멘트 컬러 통일 (작성자별 고정 색상)

**문제**: `handleAddComment`에서 `colors[Math.floor(Math.random() * 5)]`로 무작위 할당 → 동일 사용자가 매번 다른 색상

**해결**: 작성자명 해시 기반 결정적 색상 함수 `getAuthorColor()` 도입

```javascript
function getAuthorColor(authorName) {
  const colors = ['#3d8bfd', '#f59e0b', '#22d3ee', '#a78bfa', '#f472b6', '#34d399', '#fb7185', '#fbbf24'];
  let hash = 0;
  for (let i = 0; i < authorName.length; i++) {
    hash = authorName.charCodeAt(i) + ((hash << 5) - hash);
  }
  return colors[Math.abs(hash) % colors.length];
}
```

**수정 포인트**:
- `dbToComment`: DB 저장 색상 무시, 항상 `getAuthorColor(row.author_name)` 사용 → DB 마이그레이션 불필요
- `handleAddComment`: `Math.random()` → `getAuthorColor(newComment.author)`

#### 6-4. 구간 코멘트 (시간 범위)

**DB 스키마** (`schema-v3.sql`):
```sql
ALTER TABLE comments ADD COLUMN IF NOT EXISTS timecode_end_seconds REAL;
```
- NULL이면 기존 단일 포인트 코멘트
- 값이 있으면 `timecode_seconds ~ timecode_end_seconds` 구간

**타임라인 드래그 UX**:
1. 타임라인 mousedown → mousemove 시 파란색 반투명 영역으로 구간 실시간 표시
2. mouseup → 드래그 거리 < 5px이면 기존 seek 동작, ≥ 5px이면 구간 선택
3. 선택 완료 후 `pendingRange` state로 타임라인에 하이라이트 유지
4. 코멘트 입력 폼에 `00:00:01 — 00:00:32` 형식 자동 표시
5. "구간 해제" 버튼으로 단일 포인트로 복귀 가능
6. 코멘트 제출 시 `pendingRange` 자동 해제

**타임라인 마커**:
- 구간 코멘트: 점 대신 색상 범위 바 (시작~끝 영역) + 양쪽 세로선
- 단일 포인트 코멘트: 기존 점 마커 유지

**CommentItem 표시**:
- `timecodeEnd`가 있으면 `00:00:01 — 00:00:32` 형식으로 타임코드 표시

**첫 구현 후 버그픽스**:

| 문제 | 원인 | 수정 |
|------|------|------|
| 드래그 선택 후 하이라이트 즉시 사라짐 | mouseup에서 `dragStart/dragEnd`를 null로 초기화, pendingRange를 Timeline에 전달하지 않음 | `pendingRange` prop을 Timeline에 전달하여 확정 구간 별도 표시 |
| 구간 코멘트 마커가 거의 안 보임 | `${comment.color}22` (8% 투명도), `border-transparent` | 배경 `40`(25%), 활성 `55`(33%), `border-frame-accent/40`, 양쪽 끝 색상 세로선 추가 |
