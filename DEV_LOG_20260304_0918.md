# DEV_LOG — 2026-03-04

## Phase 4: 버전 관리 + 플레이어 강화

### 완료 항목

#### 4-1. DB 스키마 (supabase/schema-v2.sql)
- `project_versions` 테이블 생성 (id, project_id, version_number, vimeo_url, description, is_active, created_by, created_at)
- `comments` 테이블에 `version_id` 컬럼 추가
- RLS 정책: 멤버 CRUD + 게스트 SELECT (projects와 동일 패턴)
- 마이그레이션: 기존 프로젝트에 v1 자동 생성 + 기존 코멘트 v1 연결
- Realtime 활성화

#### 4-2. 버전 관리 API (auth.js)
- `fetchProjectVersions(projectId)` — 버전 목록 조회
- `createProjectVersion(projectId, vimeoUrl, description)` — 새 버전 추가 (자동 넘버링, 기존 active 해제)
- `switchActiveVersion(projectId, versionId)` — 활성 버전 전환

#### 4-3. 버전 관리 UI (app.js)
- `VersionTabBar` — v1 | v2 | v3 탭 바 + 추가/비교 버튼
- `AddVersionModal` — Vimeo URL + 설명 입력 모달
- `CompareView` — 좌/우 버전 선택 + 사이드바이사이드 동기 재생

#### 4-4. 사운드 컨트롤러 (app.js)
- 볼륨 슬라이더 (0~100%) + 뮤트 토글 버튼
- Vimeo SDK `player.setVolume()` 연동
- 배속 컨트롤 옆 배치

#### 4-5. 키보드 단축키 (app.js)
- Space: 재생/일시정지
- ←/→: 5초 뒤로/앞으로
- ↑/↓: 볼륨 ±10%
- M: 뮤트 토글
- F: 풀스크린 토글
- input/textarea/select 포커스 시 비활성화
- `ShortcutHelpTooltip` 컴포넌트 (키보드 아이콘 클릭 → 안내)

#### 4-6. 풀스크린 (app.js)

- 진입/해제 시 아이콘 전환
