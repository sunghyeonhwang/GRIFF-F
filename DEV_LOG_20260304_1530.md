#### 1. Vimeo 일부공개(unlisted) 영상 지원

**문제**: `vimeo.com/1170147115/99811327b3` 형식(일부공개)에서 해시가 누락되어 영상 로드 실패

**원인**: `extractVimeoId()`가 ID만 추출, Vimeo Player SDK에서 일부공개 영상은 해시가 필수

**해결**: `id` 대신 `url` 파라미터로 전체 URL을 전달
```javascript
// 변경 전
playerOpts.id = vimeoId;
// 변경 후
playerOpts.url = vimeoUrl;  // 해시 포함 전체 URL
```
- VideoPlayer, CompareView 좌/우 플레이어 모두 적용
- `extractVimeoHash()` 유틸 함수 추가

#### 3. 타임라인 진행바 + 타임코드 업데이트 안 됨

**문제**: 재생 중 seek바와 타임코드가 움직이지 않음

**원인**:
- `onTimeUpdate` 콜백이 useEffect 클로저에 갇혀 stale 상태
- `vimeoUrl` 기반으로 플레이어를 만들면서 useEffect 의존성이 `[vimeoId]`로 불일치
- 이전 iframe DOM 잔여물이 남아 플레이어 충돌

**해결**:
- 콜백 ref 패턴 도입 (`onTimeUpdateRef`, `onDurationReadyRef` 등) → stale closure 방지
- `playerKey = vimeoUrl || vimeoId`로 의존성 통일
- 플레이어 생성 전 `while (container.firstChild) container.removeChild(container.firstChild)` DOM 정리
- `timeupdate`/`ended` 이벤트를 `ready()` 콜백 내부로 이동
- `window.__griffPlayer` 항상 등록 (playerRefOut 유무 무관)

---

#### 4. 코멘트 클릭 시 영상 이동

**문제**: 코멘트 클릭이 영상 seek로 이어지지 않음

**원인**: `window.__griffPlayer`가 `playerRefOut` 존재 시 null로 남음

**해결**: `seekTo()` 공통 함수 도입, `mainPlayerRef.current || window.__griffPlayer` 우선순위로 seek
- `handleSeek`, `handleMarkerClick`, `handleCommentClick`, 키보드 단축키 모두 `seekTo()` 사용

#### 7. 키보드 단축키 한글 IME 대응

**문제**: 한글 입력 모드에서 M키(`ㅡ`), F키(`ㄹ`)가 인식되지 않음

**원인**: `e.key` 기반 매칭 — 한글 IME에서 `e.key`가 한글 자모로 들어옴

**해결**: `e.key` → `e.code` 기반으로 전환 (물리 키 위치 기반, IME 무관)
| `e.code` | 영문 | 한글 모드 `e.key` |
|-----------|------|-------------------|
| `KeyM` | M | ㅡ |
| `KeyF` | F | ㄹ |
| `Space` | Space | (동일) |
