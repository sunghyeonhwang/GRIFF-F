# DEV_LOG — 2026-03-04 11:00

## Phase 4 버그픽스: CompareView 동시 재생 문제 해결

### 문제

버전 비교 모드에서 재생 버튼 클릭 시 **한쪽 영상만 재생**되고 다른 쪽은 pause 상태로 유지됨.

### 근본 원인

**Vimeo Player SDK의 `autopause` 옵션 (기본값: `true`)**
- 한 페이지에 여러 Vimeo 플레이어가 있을 때, 한 플레이어의 `play()`가 호출되면 SDK가 내부적으로 다른 플레이어를 자동 pause 처리
- `right.play()` 호출 → SDK가 자동으로 `left.pause()` 실행

### 디버깅 과정

| 시도 | 접근 방식 | 결과 |
|------|----------|------|
| 1 | React ref로 play() 동시 호출 | 좌측만 재생 (우측 play 시 좌측 pause) |
| 2 | `muted: true`로 autoplay 정책 우회 시도 | PlayInterrupted 에러 지속 |
| 3 | 순차 체인 (left.play → right.play) | 체인 내 right.play() 시 left가 pause |
| 4 | setTimeout 200ms 딜레이 분리 | 동일 현상 |
| 5 | DOM `__vPlayer` 직접 저장 | 동일 현상 |
| 6 | `player.on('pause')` 이벤트 추적 | **right.play() 호출 시점에 left pause 이벤트 감지** |
| 7 | **`autopause: false` 옵션 추가** | **양쪽 동시 재생 성공** |

### 수정 내용
#### 문제해결 개입 
Vimeo Player API를 사용하여 두 개의 iframe 영상을 동시에 제어하려면, 각 iframe을 Vimeo Player 객체로 초기화한 뒤      
자바스크립트의 Promise.all이나 개별 play() 메서드를 호출, 프레임 단위 동기화가 필요하다면 한 플레이어의 bufferend 
이벤트를 기다린 후 재생하는 로직이 추가, 브라우저의 자동 재생 정책(Autoplay Policy) 때문에 사용자의 클릭 한          
번으로 두 영상을 동시에 틀거나, 두 영상 모두 음소거(muted) 상태여야 오류 없이 작동 이런 이야기가 있던데 이부분도  
확인요청 -> 수정적용

#### CompareView 플레이어 생성 (app.js)
```javascript
new Vimeo.Player(el, {
  // ... 기존 옵션
  autopause: false,  // 핵심 — 동시 재생 허용
});
```

#### 동기 재생 로직 개선 (app.js)
- **Promise.all**: `Promise.all([left.play(), right.play()])` 으로 양쪽 모두 성공 확인 후 시간 동기화
- **Autoplay Policy fallback**: play 실패 시 음소거(`setVolume(0)`) 후 자동 재시도
- **Drift 보정**: 2초 간격 `setInterval`로 양쪽 시간차 측정, 0.3초 초과 시 right를 left 기준으로 보정

### 학습 포인트

1. **Vimeo `autopause`**: 동일 페이지 멀티 플레이어 시 반드시 `autopause: false` 설정 필요
2. **Vimeo Player 인스턴스 관리**: container 기반 생성 후 DOM element에 `__vPlayer`로 저장하면 React 라이프사이클 간섭 없이 안정적 접근 가능
3. **Drift 보정**: Vimeo SDK는 프레임 단위 동기화를 직접 지원하지 않으므로, 주기적 polling + setCurrentTime 보정이 현실적 방법
