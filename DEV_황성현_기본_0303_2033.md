# GRIFF Frame - DEV.md

## 선택된 개발 구조

**Option 2: Supabase JS Architecture**
- 프론트엔드: HTML/CSS/Vanilla JS (빌드 도구 없음)
- 백엔드: Supabase (DB, Auth, Realtime, RLS)
- 배포: Vercel 또는 정적 호스팅
- 서버 코드 없음 - Supabase JS 클라이언트가 직접 DB와 통신

---

## Requirements

MISSION.md 기반 핵심 요구사항:

- [ ] Vimeo 영상 URL 등록 및 인앱 재생 (배속 제어 포함)
- [ ] 타임라인 기반 코멘트/피드백 (특정 타임코드에 코멘트, 마커 시각화)
- [ ] 코멘트 내보내기 (Premiere Pro XML, Final Cut Pro XML, DaVinci Resolve, CSV 선택 다운로드)
- [ ] 링크 공유 시 이름 입력 후 코멘트 가능 (회원가입 불필요)
- [ ] 내부 팀 5명 + 클라이언트 3명 규모 지원
- [ ] 프로젝트별 초대 + 링크 공유 방식 접근 제어

## Non-goals (v1 제외 범위)

- 영상 파일 직접 업로드/인코딩/호스팅 (Vimeo에 위임)
- 실시간 채팅 또는 메신저 기능
- 영상 편집 기능
- 결제/과금 시스템
- 버전 비교 (v2에서 검토)
- 프로젝트/폴더 기반 자산 관리 (v2에서 검토)
- 모바일 앱 (웹 우선)

## Style Guidelines

- UI 언어: 한국어 기본
- 디자인: 미니멀, 다크 테마 (영상 리뷰에 적합)
- 폰트: Pretendard 또는 시스템 폰트
- 타임코드 표시 형식: `HH:MM:SS:FF` (프레임 단위) 또는 `HH:MM:SS` (초 단위)
- 반응형: 데스크톱 우선 (1280px+), 태블릿 기본 지원

## Key Concepts (핵심 용어)

| 용어 | 설명 |
|------|------|
| **프로젝트(Project)** | 하나의 영상 리뷰 단위. Vimeo URL 1개 + 코멘트 모음 |
| **코멘트(Comment)** | 특정 타임코드에 연결된 피드백 텍스트 |
| **마커(Marker)** | 타임라인 위에 코멘트 위치를 시각적으로 표시하는 점/막대 |
| **타임코드(Timecode)** | 영상 내 특정 시점 (초 단위 저장, HH:MM:SS 표시) |
| **게스트(Guest)** | 링크를 통해 접근, 이름만 입력하고 코멘트 가능한 외부 사용자 |
| **멤버(Member)** | Supabase Auth로 로그인한 내부 팀원 |
| **내보내기(Export)** | 코멘트를 NLE(Non-Linear Editor) 호환 포맷으로 다운로드 |

## 외부 설정 필요 항목

| 항목 | 필요 값 | 획득 방법 | 비고 |
|------|---------|-----------|------|
| **Supabase 프로젝트** | `SUPABASE_URL`, `SUPABASE_ANON_KEY` | [supabase.com](https://supabase.com) → New Project → Settings → API | 무료 플랜으로 충분 (소규모 팀) |
| **Vimeo 계정** | Vimeo 영상 URL | 기존 Vimeo 계정 사용 | oEmbed는 API 키 불필요. Player SDK도 공개 영상은 키 불필요 |
| **Vimeo API 토큰** (선택) | `VIMEO_ACCESS_TOKEN` | [developer.vimeo.com](https://developer.vimeo.com) → My Apps → Generate Token | 비공개 영상 접근 또는 고급 Player 제어 시에만 필요 |
| **Vercel 계정** | Vercel 프로젝트 | [vercel.com](https://vercel.com) → Import Git Repository | 무료 Hobby 플랜으로 충분 |
| **도메인** (선택) | 커스텀 도메인 | Vercel 대시보드에서 연결 | 없으면 `*.vercel.app` 사용 |

### 환경변수 (.env)

```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=eyJ...your-anon-key
VIMEO_ACCESS_TOKEN=your-vimeo-token  # 선택사항
```

> **주의**: `.env` 파일은 `.gitignore`에 반드시 포함. Supabase anon key는 클라이언트에 노출되므로 RLS 정책이 보안의 핵심.

## 시작하기

### 1단계: Phase 1 프로토타입 (지금 바로 시작 가능)

```bash
# 프로젝트 폴더에서
# prototype-v1.html을 브라우저에서 직접 열기
open prototype-v1.html
```

서버도, npm도, 설치도 필요 없음. HTML 파일 하나로 시작.

### 2단계: Phase 2 개발 환경 (Phase 1 완료 후)

```bash
npm init -y
npm install express
node scripts/serve.js
# http://localhost:3000 에서 확인
```

### 3단계: Supabase 연동 (Phase 2.5)

```bash
# 1. supabase.com에서 프로젝트 생성
# 2. .env 파일 생성
echo "SUPABASE_URL=https://your-project.supabase.co" > .env
echo "SUPABASE_ANON_KEY=your-anon-key" >> .env
# 3. Supabase SQL Editor에서 supabase/schema.sql 실행
# 4. RLS 정책 적용: supabase/rls-policies.sql 실행
```

### 4단계: 배포 (Phase 4)

```bash
# Vercel CLI
npm i -g vercel
vercel
# 환경변수는 Vercel 대시보드에서 설정
```
